```
### 2. 获取归档详情
```
GET /api/v1/archives/:id
```

响应结构同列表中的单条 item。

### 3. 下载归档 zip 包
```
GET /api/v1/archives/:id/download
```

**响应：** 直接返回 `application/zip` 文件流，浏览器自动触发下载。

**zip 包内容：**
```
归档_第三季度工作会议_20260309173200.zip
├── 签到表.pdf              ← 实时生成，含签名图片
├── 投票统计.csv            ← 实时生成，含各投票选项统计和明细
├── signatures/             ← 签名原始文件
├── edited/{userId}/        ← 编辑后的文件
└── records/                ← 会议记录文件
```

### 4. 删除归档
```
DELETE /api/v1/archives/:id
响应：
json{
  "code": 200,
  "message": "删除归档成功",
  "data": null
}
5. WebSocket 归档通知（实时推送）
设备插入充电坞后自动归档，前端通过 WebSocket 监听以下消息：
type时机data 字段archive_start检测到数据开始归档device_serialarchive_complete归档完成device_serial, meeting_count, signatures_count, votes_countarchive_failed归档失败device_serial, error_message
前端收到 archive_complete 后可刷新归档列表或弹出 toast 提示。

就这 4 个 REST 接口 + 3 个 WebSocket 消息类型，归档数据的产生完全由设备插入自动触发，UI 只需要做展示列表、查看详情、下载、删除。
